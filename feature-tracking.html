What's on the dashboard

Every answer Beacon gives is scored against whether the developer marked it helpful, followed up with another question, or escalated to a human. That signal rolls up into a live resolution rate, broken down by surface, repo, and time window.

  • First-response resolution rate, updated as questions are asked
  • Flagged answers surfaced immediately, with the source they were grounded in
  • Breakdowns by channel — widget, Slack, VS Code — to see where answers land best

Why teams care

An assistant that quietly gets worse as your codebase changes is worse than one that never launched, because nobody notices until support tickets creep back up. A live number makes regressions visible the same day a docs page goes stale, instead of at the next quarterly review.

Where it fits

Post-launch monitoring Watch resolution rate right after a big API change to catch answers that went stale.
Support team reporting Pull a live number into a weekly update instead of manually tallying flagged answers.
Docs quality signal A dip in one repo's resolution rate is often the first sign its docs need attention.
