// FEATURE — SETUP

From zero to answering
questions in one sitting.

Connect a GitHub repo and a docs folder — Beacon indexes both, builds its embeddings, and is ready to answer real developer questions before the session ends.

What happens in those 5 minutes

Most "AI on your docs" tools ask for a data export, a schema mapping, and a follow-up call. Beacon skips all of it. You authorize a repo, point it at a docs folder, and indexing starts immediately — no staging environment, no separate ingestion pipeline to babysit.

  • Reads source files, markdown, and OpenAPI specs directly — no manual export step
  • Builds embeddings incrementally, so partial answers are available before indexing finishes
  • Re-indexes automatically on every push, scoped to just the changed files

Why teams care

The setup step is usually where these tools lose momentum — a week of Slack threads with an implementation engineer before anyone sees a real answer. Beacon is built so the first useful answer happens in the same sitting as the signup, while the context for why you wanted this is still fresh.

Where it fits

New API launch Point Beacon at the SDK repo the same day it ships, so support tickets have an answer on day one.
Onboarding a new engineer Skip the "ask in #eng-help" phase — index the monorepo once and let new hires query it directly.
Docs migration Mid-migration docs are half-updated. Beacon reads the code itself, so answers don't wait on the docs team.
