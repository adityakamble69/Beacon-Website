// FEATURE — CITATIONS

Every answer points back
to the line it came from.

No confident-sounding guesses about your API. Every response links to the exact file, function, or doc paragraph it was pulled from — so a dev can verify it in one click.

Answers with a paper trail

Beacon doesn't generate an answer and hope it's right — it retrieves the relevant source first, then writes the answer grounded in what it found. Each claim in the response carries a reference back to that source, shown as a small numbered chip.

  • Citations link to a specific file and line range, not just a repo or doc name
  • Hovering a citation previews the source inline, without leaving the chat
  • If Beacon can't find a grounded source, it says so instead of filling the gap

Why teams care

Trust is the actual product here. A wrong answer with no source is a support ticket in disguise; a right answer you can't verify still costs a Slack message to a teammate. Visible citations mean a developer can confirm an answer themselves, in the time it takes to read the flagged snippet.

Where it fits

Auditing AI-assisted support Support leads spot-check flagged citations instead of re-verifying every answer from scratch.
Fast-moving APIs When an endpoint changes weekly, a citation shows a dev exactly which version of the code they're reading.
Compliance-sensitive teams Every answer is traceable to a source file, which matters when "where did this come from" is an audit question.
